================================================================================

STEP 1: Install Dependencies

$ cd /Users/hikmatullo/Downloads/attendance_system_fixed
$ pip install -r requirements.txt


STEP 2: Run Application
─────────────────────────────────────────────────────────────────────────────
$ python app.py


STEP 3: Register an Employee (if needed)

   • Enter Employee Code (e.g., "EMP001")
2. Click "📷 Open Camera for Register"

STEP 4: Enable AUTO MODE
─────────────────────────────────────────────────────────────────────────────
1. In "TURNSTILE CONTROL" panel:
   • Click "🤖 AUTO MODE OFF" button
   • Button becomes "🤖 AUTO MODE ON" (green background)
   • System begins continuous scanning


STEP 5: Test Automatic Recognition
─────────────────────────────────────────────────────────────────────────────
1. Position your face in front of camera
2. Watch GUI update in real-time:
   • Status: "✅ RECOGNIZED"
   • Name: "John Smith"
   • Confidence: "98%"
   • Event Feed: "+ ✅ John Smith — IN"
   • Attendance automatically logged to database
   • Photo saved to logs/ directory


STEP 6: Stop AUTO MODE
─────────────────────────────────────────────────────────────────────────────
1. Click "🤖 AUTO MODE ON" button again
2. Button becomes "🤖 AUTO MODE OFF" (gray)

💾 DATABASE SCHEMA
================================================================================

When an attendance is logged, the following entry is created:

TABLE: logs
┌─────────────────────────────────────────────────────────────────┐
│ id          (INTEGER)  → Auto-increment log ID                 │
│ employee_id (INTEGER)  → NULL if unknown, otherwise employee ID│
│ full_name   (TEXT)     → "John Smith" or "Unknown Person"      │
│ event_type  (TEXT)     → "IN" (for attendance)                 │
│ event_time  (TEXT)     → "2025-04-07 14:32:10"                │
│ confidence  (REAL)     → 0.98 (face match score 0.0-1.0)       │
│ image_path  (TEXT)     → "logs/1_20250407_143210.jpg"          │
└─────────────────────────────────────────────────────────────────┘

Example Records:
┌────┬─────────────┬────────────────┬────────────┬────────────────────────────────────────┐
│ id │ employee_id │ full_name      │ confidence │ event_time          │ image_path      │
├────┼─────────────┼────────────────┼────────────┼─────────────────────┼─────────────────┤
│ 1  │ 1           │ John Smith     │ 0.98       │ 2025-04-07 14:32:10 │ logs/1_*.jpg    │
│ 2  │ NULL        │ Unknown Person │ 0.35       │ 2025-04-07 14:35:30 │ logs/unknown/*.jpg│
│ 3  │ 1           │ John Smith     │ 0.97       │ 2025-04-07 15:45:00 │ logs/1_*.jpg    │
└────┴─────────────┴────────────────┴────────────┴─────────────────────┴─────────────────┘

🔧 ARCHITECTURE
================================================================================

THREADING MODEL:
Main GUI Thread                 │    Background Processing Thread
─────────────────────────────────────────────────────────────────
User clicks button              │
  toggle_auto_mode()            │
    engine.start()              │
      ├─ Load employees         │
      ├─ Open camera            │
      └─ Start thread           ├──→ _run_loop() starts
                                 │
                                 ├──→ while running:
                                 │    ├─ Read camera frame
                                 │    ├─ Extract faces (every 4 frames)
                                 │    ├─ Compare encodings
                                 │    └─ Invoke callback()
                                 │
  handle_auto_result()  ←────────┤
    ├─ Update GUI labels
    ├─ Play sound
    ├─ Log to database
    └─ Refresh table

User clicks button              │
  toggle_auto_mode()            │
    engine.stop()               │
      ├─ Set running = False    ├──→ _run_loop() exits
      ├─ Release camera         │
      └─ Join thread


PERFORMANCE:
• Frame Processing: Every 4th frame (skip 3)
• Processing Speed: ~30 FPS
• GUI Response: Non-blocking (separate thread)
• Memory: Minimal (1-frame buffer, streaming processing)


================================================================================
⚙️ CONFIGURATION & CUSTOMIZATION
================================================================================

Adjustable Parameters:

1. Anti-Duplicate Time (auto_mode.py)
   ──────────────────────────────────
   ANTI_DUPLICATE_SECONDS = 30  # seconds
   
   Change to:
   • Lower (e.g., 10): Faster re-logging of same person
   • Higher (e.g., 60): Longer wait time before re-logging

2. Face Match Confidence (face_utils.py)
   ────────────────────────────────────
   CONFIDENCE_THRESHOLD = 0.50  # 0.0 (easy) to 1.0 (strict)
   
   Change to:
   • Lower (e.g., 0.35): More matches (more false positives)
   • Higher (e.g., 0.70): Fewer matches (more false negatives)

3. Frame Processing Rate (auto_mode.py)
   ────────────────────────────────────
   frame_skip = 4  # Process every Nth frame
   
   Change to:
   • Lower (e.g., 2): More frequent processing (higher CPU)
   • Higher (e.g., 8): Less frequent processing (lower CPU)
